ASP.NET Core based microservice with DDD tactical patterns inside. It is also based on Command/Query separation to provide very thin controllers with only one resposibility - calling proper query or command handler. In this way API layer is reposnsible only for wrapping logic around query and command handlers. We can say API can execute commands and queries. This layer should not reference domain itself, to not leak to the outside world. It is hence forbidden to reference Domain from API, which can reference only Commands and Queries assemblies.
